Point release with fixes and small features on top of v0.1.0. Vault files stay byte-compatible with the Android Aegis app.

## What's new

- **Double-click to copy**: double-clicking an entry copies its current code. This is the native macOS affordance, so it works regardless of the tap copy-behavior preference (even when it is set to "Never").
- **Vault password management** (Settings → Security): set a password on a plaintext vault, change the master password, or remove the password. This mirrors upstream Aegis. Encrypting a plaintext vault is what makes Touch ID available, so this is the way to add Touch ID after importing a plaintext vault.
- **Automatic Touch ID prompt**: when Touch ID is configured, the unlock screen requests it immediately instead of waiting for a "Use Touch ID" click. Cancelling falls back to the password field.

## Fixes

- **Touch ID could not store the key** ("cannot store the key in the keychain"). Ad-hoc signed builds cannot use Secure-Enclave / biometric-access-control keychain items: they get `errSecMissingEntitlement`, and are killed by AMFI if they declare the required entitlement. The vault password is now stored as an ordinary device-only login-keychain item, and Touch ID is enforced by an explicit biometric check on retrieval. This is a best-effort model for a self-signed app: the biometric prompt is enforced by the app, not bound to the key in hardware. If the password is unknown, type it instead.
- **Closing the window no longer locks the vault.** Locking now happens on app quit (the on-disk vault is always encrypted, so the next launch starts locked) and on system lock (screen lock). Reopening the window keeps an unlocked session unlocked.
- **Import is a merge.** Importing an Aegis vault file adds its entries and keeps the current vault's own encryption state, matching upstream. Vault encryption is managed separately in Settings → Security.

## Install

Requires macOS 14 (Sonoma) or later, Apple Silicon or Intel.

1. Download `AegisMac-0.1.1.dmg` below and drag **AegisMac** to Applications. It replaces any earlier version. Your vault, settings, and Touch ID entry live outside the app bundle and are preserved.
2. The app is ad-hoc signed, not notarized, so macOS blocks the first launch. Allow it under **System Settings → Privacy & Security → Open Anyway**, or clear the quarantine flag:

   ```sh
   xattr -d com.apple.quarantine /Applications/AegisMac.app
   ```

3. Optionally verify the download:

   ```sh
   shasum -a 256 -c SHA256SUMS.txt
   ```

Your vault lives at `~/Library/Application Support/AegisMac/aegis.json`, a standard Aegis vault the Android app can import directly. Back it up.

Based on upstream Aegis commit `a9d45b3` (after v3.4.2). Issues and feedback welcome on the [issue tracker](https://github.com/myl7/AegisMac/issues).
